41-缺失的第一个正数

41. 缺失的第一个正数

Difficulty: 困难

给你一个未排序的整数数组 nums ，请你找出其中没有出现的最小的正整数。

进阶：你可以实现时间复杂度为 O(n) 并且只使用常数级别额外空间的解决方案吗？

示例 1：

输入：nums = [1,2,0]
输出：3

示例 2：

输入：nums = [3,4,-1,1]
输出：2

示例 3：

输入：nums = [7,8,9,11,12]
输出：1

提示：

  • 0 <= nums.length <= 300
  • -2<sup>31</sup> <= nums[i] <= 2<sup>31</sup> - 1

Solution

原地哈希

有了时间和空间复杂度的要求之后，想来想去这道题都只能用原地哈希来做，，毕竟经验还是少，所以就尝试了一下，大概思路分为两步：

  1. 原地遍历，对原数组进行哈希，目的是将数组中的元素值进行编码，通过数组的索引体现出来（比如当前元素为3，那么就把原数组中第三个数进行哈希）。总结起来就是：通过改变原数组元素对应索引的数值状态，来编码数组中出现的元素数值信息。

    这里的哈希操作有点不同，我是将整型转换为字符串（其实也想过变化为数组，一个道理），原因如下

    原地哈希，个人理解是对原数组进行掩码，让被掩码之后的数组元素可以明显地被找出来，但是却又不能更改原数组的数据信息，前面的题有加上一个数来进行哈希的，但是那只适合元素的大小有界的情况下，所以我就想到了利用数据类型来区分。不过虽然解法通过了，但是时间复杂度和空间复杂度不确定是否满足题意。

  2. 再次遍历哈希后的数组，通过判断每个元素的类型，找到数据类型是整型的对应索引，加1之后，就是本题结果了

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        if nums == []: return 1
        length = len(nums)
        # 修改原数组，即保留原来数值，又使其具备mask信息，所以将整形转化为字符串
        for num in nums:
            x = int(num) - 1
            if x >= 0 and x < length:
                nums[x] = str(nums[x]) if isinstance(nums[x], int) else nums[x]
        # 遍历修改后的数组，判断每个元素的数据类型，遇见第一个整形的数据就返回
        for i in range(length):
            if isinstance(nums[i], int):
                return i + 1
        # 如果遍历结束也不满足return的条件，说明数组是[1, 2, 3, 4]这种的，那么最小正正数就是长度加1
        return length + 1

复杂度分析

令 n 为数组长度。

  • 时间复杂度：O(n)
  • 空间复杂度：O(1)
