495-提莫攻击

495. Teemo Attacking

Difficulty: 中等

在《英雄联盟》的世界中，有一个叫 “提莫” 的英雄，他的攻击可以让敌方英雄艾希（编者注：寒冰射手）进入中毒状态。现在，给出提莫对艾希的攻击时间序列和提莫攻击的中毒持续时间，你需要输出艾希的中毒状态总时长。

你可以认为提莫在给定的时间点进行攻击，并立即使艾希处于中毒状态。

示例1:

输入: [1,4], 2
输出: 4
原因: 第 1 秒初，提莫开始对艾希进行攻击并使其立即中毒。中毒状态会维持 2 秒钟，直到第 2 秒末结束。
第 4 秒初，提莫再次攻击艾希，使得艾希获得另外 2 秒中毒时间。
所以最终输出 4 秒。

示例2:

输入: [1,2], 2
输出: 3
原因: 第 1 秒初，提莫开始对艾希进行攻击并使其立即中毒。中毒状态会维持 2 秒钟，直到第 2 秒末结束。
但是第 2 秒初，提莫再次攻击了已经处于中毒状态的艾希。
由于中毒状态不可叠加，提莫在第 2 秒初的这次攻击会在第 3 秒末结束。
所以最终输出 3 。

提示：

  1. 你可以假定时间序列数组的总长度不超过 10000。
  2. 你可以假定提莫攻击时间序列中的数字和提莫攻击的中毒持续时间都是非负整数，并且不超过 10,000,000。

Solution

思路

  1. 提取前提条件

    题目说给定两组数据，攻击的时间点以及中毒的持续时间

    并且，时间序列以及持续时间为非负整数，所以要考虑持续时间为0的情况

    此外，给定的时间序列应该是从小到达排列的

  2. 算法思路整理

    按题意所说，我们首先确定，经过一次遍历，应该就可以得到答案，所以我们先对每个攻击时间点分析

    • 如何计算这次攻击带来的中毒时间？其实可以分为两种情况
      1. 上次中毒效果在这次攻击前结束，即不会影响到这次中毒，那么这次中毒持续时间就是完整的中毒周期duration
      2. 上次中毒效果在这次攻击开始时还未结束，那么这次攻击带来的有效中毒时间就需要从上次攻击结束时间开始算

    理清这个逻辑后，我们只需要遍历每个攻击时间点即可，然后维护两个常量，一个是上次的中毒结束时间，一个是总的有效中毒时间，最终返回有效中毒时间

class Solution:
    def findPoisonedDuration(self, timeSeries: List[int], duration: int) -> int:
        end_time = 0
        valid_time = 0
        for i in timeSeries:
            if i >= end_time:
                valid_time += duration
            else:
                valid_time += duration + i - end_time
            end_time = i + duration
        return valid_time

复杂度分析

令 n 为数组长度。

  • 时间复杂度：$O(n)$
  • 空间复杂度：$O(1)$
