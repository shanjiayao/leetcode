274-H-指数

274. H 指数

Difficulty: 中等

给定一位研究者论文被引用次数的数组（被引用次数是非负整数）。编写一个方法，计算出研究者的 h 指数。

：h 代表“高引用次数”（high citations），一名科研人员的 h 指数是指他（她）的 （N 篇论文中）总共有 h 篇论文分别被引用了至少 h 次。且其余的 N - h 篇论文每篇被引用次数 不超过 h 次。

例如：某人的 h 指数是 20，这表示他已发表的论文中，每篇被引用了至少 20 次的论文总共有 20 篇。

示例：

输入：citations = [3,0,6,1,5]
输出：3
解释：给定数组表示研究者总共有 5 篇论文，每篇论文相应的被引用了 3, 0, 6, 1, 5 次。
     由于研究者有 3 篇论文每篇 至少 被引用了 3 次，其余两篇论文每篇被引用 不多于 3 次，所以她的 h 指数是 3。
  • 提示：**如果 h 有多种可能的值，h* 指数是其中最大的那个。

Solution

本题其实难点还是在于理清思路，解法如下：

首先假设一个长度为N的列表，列表里每一个数字表示一篇论文的引用次数，为了更好地理清思路，我们先对列表进行升序排序，如下：

0， 1， 2， 3， 4， 5， 6， ...， N

下划线的元素表示有h篇论文，并且所有论文的引用次数都大于或等于h，那我们只需要判断下划线的第一个元素是否大于或等于h即可

进一步地，我们可以把这个条件和数组遍历结合起来，从尾端遍历数组，h从1开始，每更换一个元素，h加1，然后计算对应的元素是否大于等于h，以及左侧元素是否小于等于h，满足条件的话即可返回。

最后，我们要考虑几种特殊情况：

  • 列表中所有元素都不大于h，比如都为0，这时说明h等于0，，可能是写的论文太垃圾没人引用吧
  • 列表中所有元素都大于列表长度，那么h就等于列表长度
class Solution:
    def hIndex(self, citations: List[int]) -> int:
        length = len(citations)
        citations.sort(reverse=False)

        for i in range(length):
            cur_h = i + 1
            cur_cita_num = citations[length - cur_h]

            # 当遍历到数组最后一个元素的时候，说明在之前都没有满足返回条件。可能的情况有两种
            # - citations所有元素都大于当前h，不满足左侧元素小于当前h的条件
            # - citations所有元素都小于当前h，不满足当前引用数量大于当前h的条件
            if i == length - 1:
                return cur_h if cur_cita_num >= cur_h else 0
            elif i < length - 1:
                left_cita_num = citations[length - cur_h - 1]
                if left_cita_num <= cur_h <= cur_cita_num:
                    return cur_h

复杂度分析

令 n 为数组长度。

  • 时间复杂度：O(nlogn) 排序
  • 空间复杂度：O(n)
